Subscribe an endpoint on the Webhooks page to receive a signed POST whenever a subscribed event happens.

Events

EventFires when
message.sentA message you sent was handed off to the mail server successfully.
message.failedSending a message failed after retries were exhausted.
message.openedThe recipient opened the email (tracking pixel).
message.clickedThe recipient clicked a tracked link.

message.delivered and message.bounced aren't available yet — "sent" means handed off to the mail server, not confirmed delivery to the recipient's inbox.

Payload

POST https://yourapp.com/webhooks/akilent
Content-Type: application/json
X-Akilent-Event: message.sent
X-Akilent-Signature: t=1730000000,v1=5257a869e7...

{
  "event": "message.sent",
  "data": {
    "id": 4821,
    "from": "billing@yourdomain.com",
    "to": "customer@example.com",
    "subject": "Payment received",
    "status": "sent"
  }
}

Verifying the signature

X-Akilent-Signature is t=<unix timestamp>,v1=<hex HMAC-SHA256> computed over "<timestamp>.<raw request body>", using your endpoint's signing secret (shown on the Webhooks page). Always verify against the raw bytes, before any JSON parsing. 

import hashlib
import hmac
import time

def verify_akilent_signature(secret: str, raw_body: bytes, signature_header: str, tolerance: int = 300) -> bool:
    parts = dict(p.split("=", 1) for p in signature_header.split(","))
    timestamp = int(parts["t"])
    received = parts["v1"]

    if abs(time.time() - timestamp) > tolerance:
        return False  # too old — possible replay

    signed_payload = f"{timestamp}.".encode() + raw_body
    expected = hmac.new(secret.encode(), signed_payload, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, received)

Retries

A non-2xx response (or a connection failure/timeout) is retried with exponential backoff, up to several attempts over roughly an hour, before the delivery is marked exhausted. Failed and exhausted deliveries can be resent manually from the Webhooks page.
