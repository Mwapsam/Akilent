Templates let you save a reusable subject/text/html body once and reference it by ID or slug from messages and campaigns, instead of repeating the content on every request. Requires a plan with the templates feature and an API key with the templates:manage scope (see Scopes).

List templates

GET /api/v1/templates
Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx
200 OK
[
  {
    "id": 12,
    "name": "Payment receipt",
    "slug": "payment-receipt",
    "subject": "Payment received",
    "updated_at": "2026-06-02T10:15:00Z"
  }
]

This list is not paginated — it returns every active template for your account as a single array. If you maintain a very large template library, be aware there's currently no page size or cursor parameter.

Create a template

POST /api/v1/templates
Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx
Content-Type: application/json

{
  "name": "Payment receipt",
  "slug": "payment-receipt",
  "subject": "Payment received",
  "text": "Hi {% templatetag openvariable %} customer_name {% templatetag closevariable %}, thanks for your payment.",
  "html": "<p>Hi {% templatetag openvariable %} customer_name {% templatetag closevariable %}, thanks for your payment.</p>",
  "sample_variables": { "customer_name": "Ada" }
}
FieldRequiredNotes
nameYesUp to 150 characters.
slugNoUp to 150 characters; used to reference the template from campaigns. Auto-derived if omitted.
subjectNoDefaults to empty.
textNoPlain-text body.
htmlNoHTML body.
sample_variablesNoA dict of example values, used for previewing the template — not required for sending.

subject, text, and html are rendered as Django template syntax — {% templatetag openvariable %} name {% templatetag closevariable %} for variable substitution, plus the built-in template tags and filters ({% templatetag openblock %} if {% templatetag closeblock %}, |default, etc.). Rendering is sandboxed to only the variables you pass at send time — it never has access to your account data, settings, or the request. 

201 Created
{
  "id": 12,
  "name": "Payment receipt",
  "slug": "payment-receipt"
}

Get, update, or delete a template

GET    /api/v1/templates/<slug>
PATCH  /api/v1/templates/<slug>
DELETE /api/v1/templates/<slug>

GET returns the full template, including text, html, and sample_variables alongside the list fields. PATCH is partial — only the fields you include in the request body are updated. DELETE returns 204 and is a soft delete: the template is deactivated (is_active=False) rather than permanently removed, and will no longer appear in list results or be usable by slug.

Errors

StatusCodeCause
403permission_deniedEither your plan doesn't include email templates, or the API key is missing the templates:manage scope — check the message field to tell which.
404not_foundNo active template with that slug on your account.
