Versioning

The REST API is versioned by URL prefix — everything documented here lives under /api/v1/. There is no v2 yet; requesting an unknown version returns a plain 404 (not the standard error envelope, since it never reaches a versioned view).

When a v2 ships, v1 will keep working unchanged — breaking changes are shipped as a new version, not as in-place edits to an existing one. Additive, backward-compatible changes (a new optional field, a new endpoint) can land on v1 directly and will be logged below.
