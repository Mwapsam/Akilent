If you'd rather send through your own mail library or an existing SMTP integration instead of calling the HTTP API, provision a relay credential and point it at Akilent's SMTP endpoint.

1. Provision a credential

On the Domains page, each verified domain has an "SMTP relay" section — click Create SMTP relay credential. The password is shown once; copy it immediately, it can't be retrieved again (only rotated or revoked).

Credentials are scoped to one verified domain each — a credential for mail.yourdomain.com can't send as a different domain your account also owns, even if that domain is verified too. Provision a separate credential per domain you want to relay from.

2. Connect

Host:     {{ smtp_relay_host }}
Port:     {{ smtp_relay_port }}
Security: STARTTLS
Username: relay@yourdomain.com   (shown when you create the credential)
Password: (the one-time secret from step 1)

3. Send

From here it's standard SMTP AUTH — configure your language's mail library (Python's smtplib, Node's nodemailer, PHP's PHPMailer, etc.) with the host/port/username/password above. The From address on messages you send must belong to the credential's domain.

Rotating or revoking

Use Rotate to issue a new password without changing the username (update your configured secret afterward), or Revoke to disable the credential entirely. Both take effect immediately.
