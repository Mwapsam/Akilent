API keys

Generate a key from the Domains page. Keys look like ak_live_… and are shown to you once, at creation — Akilent stores only a hash, not the plaintext, so if you lose it you'll need to regenerate rather than retrieve it.

Send it on every request as either header:

X-Api-Key: ak_live_xxxxxxxxxxxxxxxxxxxxxxxx
# or
Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx

Regenerating a key immediately deactivates the previous one — there's no overlap window today, so update all callers before you rotate.

A missing or invalid key returns 401 with {"error": {"code": "authentication_failed", ...}}. Repeated invalid attempts from the same client are temporarily locked out (429) as a brute-force guard.

Scopes

Beyond the plan-tier feature gates (does your plan include the API, bulk email, templates?), individual API keys carry a scopes list that independently controls which endpoints that specific key can call.

ScopeRequired by
messages:sendNothing today — POST /api/v1/messages doesn't check scopes. This is the default scope on every new key.
templates:manageAll of the templates endpoints.
messages:send:bulkBoth campaigns endpoints.

Every key created from the Domains page today defaults to ["messages:send"], and there's no self-service UI yet to grant a key additional scopes. If you need a key that can manage templates or launch campaigns, contact support to have the scope added — don't assume a key can call an endpoint just because your plan includes the feature; the key also needs the matching scope, or you'll get a 403 permission_denied.

SMTP relay credentials

Separate from API keys — see SMTP relay for how to provision one. These authenticate an SMTP connection directly against the mail server, not calls to /api/v1/, and scopes don't apply to them.
