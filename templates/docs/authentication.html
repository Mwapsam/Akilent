API keys

Generate a key from the Domains page. Keys look like ak_live_… and are shown to you once, at creation — Akilent stores only a hash, not the plaintext, so if you lose it you'll need to regenerate rather than retrieve it.

Send it on every request as either header:

X-Api-Key: ak_live_xxxxxxxxxxxxxxxxxxxxxxxx
# or
Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx

Regenerating a key immediately deactivates the previous one — there's no overlap window today, so update all callers before you rotate.

A missing or invalid key returns 401 with {"error": {"code": "authentication_failed", ...}}. Repeated invalid attempts from the same client are temporarily locked out (429) as a brute-force guard.

SMTP relay credentials

Separate from API keys — see SMTP relay for how to provision one. These authenticate an SMTP connection directly against the mail server, not calls to /api/v1/.
