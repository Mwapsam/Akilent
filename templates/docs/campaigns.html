Campaigns send one message to many recipients in a single call, each optionally personalized with per-recipient variables. Requires a plan with bulk email sending and an API key with the messages:send:bulk scope (see Scopes).

Launch a campaign

curl -X POST https://api.akilent.com/api/v1/campaigns \
  -H "Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "from": "billing@yourdomain.com",
    "template_id": 12,
    "recipients": [
      { "to": "ada@example.com", "variables": { "customer_name": "Ada" } },
      { "to": "grace@example.com", "variables": { "customer_name": "Grace" } }
    ]
  }'
import requests

resp = requests.post(
    "https://api.akilent.com/api/v1/campaigns",
    headers={"Authorization": "Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx"},
    json={
        "from": "billing@yourdomain.com",
        "template_id": 12,
        "recipients": [
            {"to": "ada@example.com", "variables": {"customer_name": "Ada"}},
            {"to": "grace@example.com", "variables": {"customer_name": "Grace"}},
        ],
    },
)
resp.raise_for_status()
print(resp.json())
FieldRequiredNotes
fromYesMust be on a verified sending domain for your account.
template_idSee noteID of a saved template.
subject / text / htmlSee noteInline content, as an alternative to template_id.
recipientsYesA list of {"to": "<email>", "variables": {...}} objects. variables is optional per recipient.

template_id and inline subject/text/html are mutually exclusive alternatives, not additive — provide one or the other. Omitting both returns a missing_content error.

Response

Launching a campaign queues it and returns immediately:

202 Accepted
{
  "id": 88,
  "status": "queued",
  "recipient_count": 2
}

Check progress

GET /api/v1/campaigns/88
Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx
200 OK
{
  "id": 88,
  "status": "sending",
  "recipient_count": 2,
  "queued_count": 0,
  "sent_count": 1,
  "failed_count": 0
}

Poll this endpoint for progress, or subscribe to webhooks — each recipient in a campaign fires its own message.sent / message.failed event, same as a single message send.

Recipient caps and volume

Your plan caps how many recipients a single campaign can target; exceeding it returns recipient_cap_exceeded before anything is queued. Each recipient in an accepted campaign counts individually against your monthly send volume, the same as if you'd called /api/v1/messages once per recipient.

Errors
