Two independent limits apply to /api/v1/messages — hitting either one gets rejected, but for different reasons and with a different status code.

Request rate (per minute)

Each API key is limited to a number of requests per minute, set by your plan (and overridable per-key). Exceeding it returns: 

429 Too Many Requests
Retry-After: 42

This protects against bursts and misbehaving clients — it resets every minute and has nothing to do with how many emails you're allowed to send this month.

Monthly send volume

Separately, your plan caps how many emails you can send per calendar month. Reaching it returns: 

403 Forbidden
{
  "error": {
    "code": "emails",
    "message": "Monthly email limit of 10000 reached. Please upgrade your plan."
  }
}

This is reserved at accept time — a message that's queued counts against the cap immediately, and if it ultimately fails to send, the reservation is refunded so you aren't charged for mail that never went out.

SMTP relay

Neither limit above applies to SMTP relay — the mail server accepts and delivers those messages directly, without passing through Akilent's application layer, so today they aren't counted against your monthly volume cap or reflected in the dashboard's usage numbers. Treat SMTP relay as currently unmetered; this will change in a future update.
