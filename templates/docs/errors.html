Every error response from /api/v1/ uses the same envelope:

{
  "error": {
    "code": "unverified_domain",
    "message": "'yourdomain.com' is not a verified sending domain for this account"
  }
}

Always branch your error handling on code, not the human-readable message text — messages may be reworded, codes won't change without a version bump (see Versioning & changelog).

StatusCodeMeaning
400invalidRequest validation failed — check the message for which field.
400missing_contentA template or campaign request had neither a template_id nor inline subject/text/html.
401authentication_failedMissing or invalid API key.
403unverified_domainThe from address's domain isn't verified for your account.
403permission_deniedThe request was authenticated but not allowed — either your plan doesn't include a feature you're using (API & SMTP relay, templates, or bulk email), or your API key is missing a required scope. Read the message field to tell which.
403emailsMonthly email send volume cap reached — upgrade or wait for the next billing period.
403recipient_cap_exceededA campaign's recipient list exceeds your plan's per-campaign cap.
404not_foundThe requested template or campaign doesn't exist on your account.
404—Unknown API version (e.g. requesting /api/v2/... before it exists).
429—Per-key request-rate limit exceeded. Check Retry-After.

See Rate limits for the difference between the 429 (rate) and 403 emails (volume) cases — they're two separate limits.
